# Local-First Conf 2026 transcripts

Transcripts and summaries of recorded talks from [Local-First Conf 2026](https://www.localfirstconf.com/) in Berlin.

## Sunday, July 12

### Local-first in an unstable world

[Summary](<transcripts/summaries/2026-07-12 - Martin Kleppmann - Local-first in an unstable world.md>) | [Transcript](<transcripts/cleaned/2026-07-12 - Martin Kleppmann - Local-first in an unstable world.md>)

**Martin Kleppmann** argues that local-first software can contribute to technological sovereignty as geopolitical instability makes dependence on centralized, foreign-controlled cloud services a societal risk.

### Agents on the canvas

[Summary](<transcripts/summaries/2026-07-12 - Steve Ruiz - Agents on the canvas.md>) | [Transcript](<transcripts/cleaned/2026-07-12 - Steve Ruiz - Agents on the canvas.md>)

**Steve Ruiz** presents the canvas as a high-bandwidth environment where people and AI agents can work together, especially when local execution gives agents capabilities that would be unsafe in a hosted service.

### Local-first from the factory floor

[Summary](<transcripts/summaries/2026-07-12 - Jeremy Theocharis - Local-first from the factory floor.md>) | [Transcript](<transcripts/cleaned/2026-07-12 - Jeremy Theocharis - Local-first from the factory floor.md>)

**Jeremy Theocharis** explains how factory constraints led to a local-first industrial data platform that keeps its open-source data plane on customer infrastructure while providing a remote management console.

### Building more resilient local-first software with atproto

[Summary](<transcripts/summaries/2026-07-12 - Jake Lazaroff - Building more resilient local-first software with atproto.md>) | [Transcript](<transcripts/cleaned/2026-07-12 - Jake Lazaroff - Building more resilient local-first software with atproto.md>)

**Jake Lazaroff** shows how local-first applications can use shared atproto infrastructure for identity, sync, and discovery instead of depending on bespoke servers whose disappearance would break the product.

### Beyond Git: Real-Time Version Control for Godot

**Lilith Duncan** presents Backstitch, an Automerge-based Godot plugin that brings real-time, domain-aware version control to game projects without forcing collaborators through Git's text-oriented workflow.

### npmx: a fast, community-built browser for the npm registry

[Summary](<transcripts/summaries/2026-07-12 - Matias Capeletto and Willow Ghost - npmx - a fast, community-built browser for the npm registry.md>) | [Transcript](<transcripts/cleaned/2026-07-12 - Matias Capeletto and Willow Ghost - npmx - a fast, community-built browser for the npm registry.md>)

**Matias Capeletto and Willow Ghost** present npmx as both a community-built alternative browser for the npm registry and an experiment in durable governance, shared ecosystem data, and connections between open-source communities.

### Plaintext-first apps in the age of agents

**Tristan Edwards** argues that local coding agents make plaintext-first software a practical foundation for powerful AI workflows, illustrated by an open-source calendar built around local ICS files.

### Frontier LLM results, on device

**RL Nabors** lays out a measurement-driven process for prototyping an AI feature with a frontier model and then finding the smallest good-enough model that can run privately on the user's device.

### Your Own AI Infrastructure: Building a Complete Local AI Stack

**Confidence Okoghenun** presents a practical four-layer architecture for running a private AI service locally using suitable hardware, an inference engine, model orchestration, and an API gateway.

### Local-first Matrix

### Panel: Data Ownership Beyond Local First

[Summary](<transcripts/summaries/2026-07-12 - Eileen Wagner, Robin Berjon, Paul Frazee, Boris Mann, and Zenna Fiscella - Panel - Data Ownership Beyond Local First.md>) | [Transcript](<transcripts/cleaned/2026-07-12 - Eileen Wagner, Robin Berjon, Paul Frazee, Boris Mann, and Zenna Fiscella - Panel - Data Ownership Beyond Local First.md>)

**Eileen Wagner, Robin Berjon, Paul Frazee, Boris Mann, and Zenna Fiscella** explore how meaningful data ownership depends not only on possessing data, but also on usable capabilities, shared infrastructure, standards, governance, and bridges between communities and protocols.

### Cueport: A local-first music player for DJs

### How the Desktop UX needs to evolve to keep up with Local First

**Scott Jenson** proposes three incremental ways desktop UX could better support a local-first future: spatial window management, persistent work areas, and privacy-conscious activity history.

## Monday, July 13

### Solving for scale in open networks

### Mosaic: Scalable interactive data visualization for the browser

### Towards a Local-First Desktop

**Tobias Bernard and Andreas Dzialocha** describe an effort to make encrypted local-first synchronization a native part of the Linux desktop so independent applications can share identity, device, networking, and sync infrastructure.

### Iroh: syncing terabytes of data, peer-to-peer

**Brendan O'Brien and Rae McKelvey** trace Iroh's evolution into a focused open-source networking library that uses cryptographic keys, multipath QUIC, hole punching, and optional relays to connect devices reliably.

### Local Digital Objects using Zero-knowledge Cryptography

**Rob Knight** shows how recursive zero-knowledge proofs, a logic language, and Merkle-addressed data can make local digital objects verifiable and shareable without exposing their private contents or relying on a central authority.
